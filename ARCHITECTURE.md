### Важно: совмещение ролей
Один человек может иметь несколько ролей одновременно:
- Старший мастер + преподаватель

## 📚 БЛОК 1: Онлайн-обучение

### Структура контента
```
Курс (привязан к ролям — можно несколько ролей на один курс)
  └── Модуль
        └── Урок
              ├── Текст
              ├── Видео (Яндекс.Облако, presigned URL 2ч)
              ├── Комментарии (заложено в БД, UI — v2)
              ├── Тест (порог 95%, пересдача до победы)
              └── Практическое задание (текст, мин. 50 слов, ИИ-проверка)
```

### Правила прохождения
- Строго по порядку
- Онлайн-блок (анатомия, физиология) — пройти ДО первого практического занятия
- Не блокирует практику если тест не сдан
- Повторное прохождение разрешено при активной подписке
- После завершения — доступны «Повторение» и «Новые материалы»

### Контент-менеджмент
- Загружают: superadmin + manager
- Черновики — публикация вручную
- Drag & drop порядка модулей и уроков
- При редактировании — система спрашивает «сбросить прогресс?»
- При удалении — прогресс архивируется, не удаляется
- При тотальном обновлении — все проходят заново

## 🎓 БЛОК 2: Академия новичков

> Только для сети Rehab.You в v1.
> В архитектуре заложена возможность продажи другим студиям.

### Программа обучения новичка

**Этап 1 — до 2 недель (базовый допуск):**
- Классический массаж
- Спортивный массаж
- Релакс массаж
- Сервис (онлайн-блок)
- Стандарты (онлайн-блок)

**Этап 2 — до 1,5 месяца (полный допуск):**
- Антицеллюлитный массаж
- Массаж лица
- Тейпирование
- Стоун-массаж (камни)

**Финал — аттестация:**
- Практическая сдача: спортивный + релакс массаж
- Принимает: superadmin / senior_master / teacher (manager — не может)
- После — фиксируется статус и список допущенных услуг

### Расписание занятий
- Преподаватель создаёт: дата, время, тема, мин. кол-во учеников
- Мин. учеников по умолчанию = 1, задаёт superadmin
- Если мин. = 1 — нужно подтверждение superadmin или manager
- Записываются: новички (обязательно) + старые мастера (добровольно)
- Занятие отменено → автоуведомление всем записавшимся

### Посещаемость и пропуски
- Предупреждение об отсутствии: минимум за 24 часа
- Предупредил за 24ч+ → не считается пропуском
- Счётчик cumulative за всё время

| Пропусков | Действие |
|---|---|
| 2 | Уведомление старшему мастеру + управляющей |
| 3 | Автоблокировка |

- Откат блокировки: manager / owner / superadmin → audit_log

### Оценка качества
- Числовая шкала 1–10 + чеклист критериев
- Шаблоны критериев — отдельные для каждого вида массажа
- Надиктовывает @drleden

### Журнал прогресса новичка (шаблон)
- Дата, тема занятия, кол-во часов обучения
- Оценка по критериям + итоговый балл
- Общий комментарий
- Дата приёма на работу
- Результат собеседования
- Результат психологического теста

### Аттестация

| Попытка | Условие |
|---|---|
| 1-я | Новичок запрашивает сам |
| 2-я | Не позднее 5 дней после 1-й |
| 3-я | Не позднее 5 дней после 2-й |
| 4-я+ | Только с разрешения superadmin / owner / manager → audit_log |

- Дату подтверждает: manager / senior_master / teacher / superadmin / ассистентка
- Провал всех попыток без разрешения = увольнение

### Статусы новичка
- `in_training` — в процессе
- `base_certified` — допущен к классике, спорту, релаксу
- `full_certified` — прошёл всю программу
- `blocked` — три пропуска
- `failed` — провалил аттестацию

## 🤖 ИИ-функциональность

### 1. Проверка практических заданий
- Минимум 50 слов
- ИИ проверяет: соответствие теме, осмысленность, не случайный набор слов
- Статусы: принято / отклонено (с комментарием почему)
- Отклонённое задание → мастер видит причину и может переотправить

### 2. ИИ-ассистент для управляющей и owner
- Чат внутри платформы (не в Telegram)
- Контекст: только функционал платформы обучения
- Что умеет:
  - Ответить на вопросы по статистике обучения («кто из мастеров не заходил?»)
  - Напомнить о pending-задачах («у вас 2 неотвеченных вопроса от мастеров»)
  - Объяснить результаты тестов
  - Дать рекомендации по обучению («Иван застрял на модуле 2 — вот что можно сделать»)
- История чатов сохраняется в БД

### 3. ИИ-сводка для superadmin и owner
- **По запросу:** команда в чате или кнопка в интерфейсе
- **По расписанию:** время и частота настраиваются вручную (по умолчанию — пн, 9:00)
- Содержание сводки:
  - Кто прошёл / не прошёл курсы за период
  - Кто не заходил и как давно
  - Результаты тестов и аттестаций
  - Проблемные зоны (кто застрял, кто на грани блокировки)
  - При наличии данных из Yclients — корреляция обучения и показателей работы
- Формат: структурированный текст с выводами и рекомендациями

### 4. Интерпретация психологических тестов
- ИИ генерирует описание результата под конкретную должность
- Структура: сильные стороны / зоны роста / рекомендации для руководителя
- Фиксируется в карточке пользователя сразу после теста

**Статус:** заложено, реализуется после запуска v1

**Что получаем (v1):**

- Количество клиентов

**Где используется:**
- В профиле мастера — показатели работы рядом с прогрессом обучения

- В аналитике superadmin / owner

**Связь с аналитической системой маркетолога:**
- analytics.rehabyou.site — отдельный проект, отдельная кодовая база

- В БД заложены поля `yclients_staff_id` для будущей связки

**Что отдаём (v2):**
- Статус обучения мастера (для отображения в Yclients)

**Статус:** заложено, реализуется после запуска v1

**Статус:** реализуется в v1

## 💳 Подписки и оплата

### Тарифы
- До 10 человек — фиксированная цена
- Свыше 10 — фикс + за каждого сверх пропорционально

### Триальный период
- 7 дней бесплатно, только первый модуль каждой роли
- После триала без оплаты — доступ заморожен, данные не удаляются

### Промокоды
- Одноразовые, дата истечения задаётся вручную superadmin'ом

### Напоминания об оплате
- За 7 дней, 3 дня, 1 день → owner
- В день окончания — автозаморозка
- Прогресс восстанавливается автоматически после оплаты

## 📊 Аналитика

**Онлайн-обучение:**
- Среднее время прохождения урока
- На каком тесте чаще застревают
- Конверсия: начали → завершили
- Кто не заходил и как давно

**Академия:**
- Посещаемость по каждому мастеру
- Средняя оценка преподавателей (анонимно)
- Прогресс новичков по этапам
- Статистика аттестаций
- Среднее время от найма до полного допуска

**Кросс-аналитика (после интеграции с Yclients):**
- Корреляция: прошёл курс → изменение возвращаемости
- Корреляция: аттестован → изменение выручки
